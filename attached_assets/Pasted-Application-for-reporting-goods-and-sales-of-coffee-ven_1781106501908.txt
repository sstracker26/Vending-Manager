Application for reporting goods and sales of coffee vending machines, and vending machines for snacks, waffles and drinks.
There should be 2 panels – user and administrator (protected with passwords for Administrator and Moderator).
The admin panel should have:
1) Dashboard with statistics - statistics for each client (what machines are there, how many products are sold by each machine, revenues, expenses, profit). Statistics for the implementation of the customer service plan. Charts for the 5 best-selling products and the 5 worst-selling products. There should also be options for filtering by date/time period for all charts and statistics in this menu,.
2) Menu for defining clients (commercial establishments/vending centers) – for example, VioModa, Pekarnata, Zavoda, Energoto, etc. For each client it should be possible to define what machines and how many are installed (choose from those defined in the Machines menu). Machine numbering – unique number, different for each machine for each client. 
Fields for the presence of a contract and start date. Option for import/export in Excel
3) Menu for defining machines. The machines are of several different types – Vending coffee machines, and Vending machines. Option for import/export in Excel
a) Vending coffee machines can be of different brands, and with different numbers of containers (with 5 containers, with 6 containers, with 7 containers, etc.). For each machine, it should be possible to specify what brand and model, and how many containers it has (one is always for coffee beans, the rest are for instant hot drinks)
b) Vending machines for snacks, waffles and drinks – for them it should be possible to specify how many rows they have, and how many chutes (one row can have a different number of chutes)
4) Menu for defining products. Products are also divided into types - for coffee machines and for vending machines (similar to the specified machine types in the Machines menu. Each product must have a name, purchase and sale price. The program must be able to keep a record of product availability - the admin will add products and put them in stock (respectively, they will enter stock with a + sign), and the operator will load the machines with these products (respectively, they will leave stock with a - sign). Units of measurement - kilogram, liter or number. Accordingly, if it is a number, it is only an integer, if it is a liter or kilogram, it can be up to 2 digits after the decimal point. Option for import/export in Excel
Option for official entry/exit of a product (from the warehouse, from a client's machine, due to expired date, defect, etc.). Official entry into a client's machine should not generate a sale, only remove it from stock in the warehouse, official exit (regardless from where, to be treated as an expense.
5) Report menu – filters by objects, by machines, by products, calendar for date/period. Option to export to Excel according to the specified criteria
6) Operator definition menu – name, operator type, if marked as admin – password field. 3 types of operators – ordinary operator, Moderator and Admin.
a) Admins have full access and full rights in all menus, they can enter the admin menu with their own password. The login panel is common, there is no need to write a user, only a password, the application only recognizes who has logged in depending on the entered password. An admin does not have the right to edit another admin's password, only their own. Keep logs of which admin changed when and what.
b) Moderators must have a separate password (one for all, set and edited by the admin), can see the menus (without the Logs menu), but without the right to edit/add/delete/import/export clients, machines, schedules and operators.
c) The ordinary operator does not have access to the admin panel, and does not need a password for the operator panel
d) Define a master password that gives the logged in user full access to absolutely everything, including changing passwords for each admin. Without a visible option for changing from the application, it can only be changed by the developer in the Replit environment.
7) Schedule definition menu – which client is served on which day of the week. The execution of the schedules is reset every Sunday evening. Application time zone Greenwich +2 (time zone for Sofia, Bulgaria)
8) Change logs menu made by admins – a general list of changes, starting from the most recent, filter by date/period
Profit is considered the difference between the purchase and sale price, and there must be an option for additional expenses that reduce profit - electricity, rent, repairs, salary, etc. It would probably be good to set these additional costs for each client separately, as rent and electricity are usually constant and rarely change, while the remaining additional costs are partly unplanned, and should be able to be entered at any time 
The operator menu (home page of the application, without the need for login) should have: 
• A field for selecting a client (drop-down menu with an option to predict when entering the first letters) 
• A subsequent field for selecting a machine (depending on the client and the machines defined for him) If possible, generate a QR code in the admin menu, Customers section, for each individual machine at the client, this code should be printed and stuck on the machine, so that when scanned with the camera, it directly opens the operator menu in the browser with these first two fields (client and machine) filled in 
• A field for selecting products. 
• Operator selection field (drop-down menu with prediction option when entering the first letters) The operator selects which product and how many pieces/kg he has placed in the machine, and this product is removed from the warehouse. By assumption, each machine is pre-loaded with n number of products when assigned to a client, respectively, if the operator adds, for example, 5 products to the machine, this means that exactly that many were sold before. This is how sales are counted, i.e. the number of products entered by the operator is equal to the number of products sold.
 Second menu in the operator panel, with visualization of the implementation of the customer service plan (from the Dashboard), only with a filter by dates/periods The menu should be located on the left side, and in the operator panel it should be hidden automatically, and there should be a button to show it, or it should be shown automatically when you place the mouse cursor on the far left.
Date format in the application DD.MM.YYYY, time format HH:MM
